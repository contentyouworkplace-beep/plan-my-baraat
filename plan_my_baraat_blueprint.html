Visual Identity & Design Language

The visual aesthetics of PlanMyBaraat must evoke the sensory richness, scale, and luxury of Indian wedding traditions. To ensure a coherent feeling across desktop browsers and mobile screens, we employ the following design system:

1. The Premium Color Palette

Bridal Crimson
#7C1C2B
Royal Gold
#B8860B
Emerald Teal
#0B4F35
Imperial Ivory
#FCFBF9
Champagne
#F5EBE0
  • Bridal Crimson (#7C1C2B): Symbolizes bridal attire, energy, and warmth. Used as the main color for headings, primary buttons, and luxury branding anchors.
  • Royal Gold (#B8860B): Symbolizes brass band instruments, palace gateways, and heritage. Used for accents, borders, star ratings, and visual trims.
  • Emerald Teal (#0B4F35): Symbolizes jade gemstones and traditional success colors. Used for success states, active vendor tags, and confirm prompts.

2. Font Pairing & Typography

  • Heading Font: Cinzel (Google Fonts)
    Aesthetic: High-contrast, elegant serif proportions modeled after classic Roman lettering. Convey immediately that the platform deals with premium heritage and palace weddings.
  • Body & UI Font: Outfit (Google Fonts)
    Aesthetic: Geometric, clean sans-serif with excellent vertical rhythm. Ensures that dense CRM tables, pricing fields, and mobile forms are highly legible.

3. Visual Accents & Component Styling

  • Glassmorphism (Frosted Glass): Headers and navigation menus use `backdrop-filter: blur(12px) bg-white/90` with gold card borders to create spatial depth.
  • Gradients: Transition backgrounds are defined using a custom linear gold gradient (`linear-gradient(135deg, #b8860b 0%, #dfc282 50%, #8c6200 100%)`).

Part II: Web Application Specifications
PlanMyBaraat

Website & CRM Web Specification

The Next.js workspace houses two major systems: the public discovery directory for clients, and the operations command dashboard for admins (with all calculator functionalities excluded).

1. Public Website Directory (planmybaraat.com)

🌐 VISUAL MOCKUP: public_website_catalog_grid
  • Dynamic Directory Search: Fast index page allowing queries across all 53 categories in target cities (Delhi-NCR, Mumbai, Ahmedabad, Surat, Vadodara, and Bengaluru).
  • Dynamic Wishlist: Bottom sliding drawer saving selected vendor cards into local state and syncing with client mobile databases.
  • WhatsApp Inquiry Form: Quick modal allowing customers to send inquiries directly, which bypasses simple email routing and logs raw data into the operations CRM.
  • Bespoke Review Submission: Form for clients to add candid star ratings and reviews which instantly recalculates average vendor scores.

2. Operations CRM Dashboard (planmybaraat.com/crm)

📊 VISUAL MOCKUP: crm_admin_pipeline_kanban
  • Leads Pipeline (Kanban Board): 5-column pipeline (New, Contacted, In Negotiation, Booked, Lost). Cards contain names, phone numbers, value, and notes. Admins can single-click to shift columns.
  • Vendor Verification Hub: Verification queue displaying pending business signups. Admins can audit documents and click "Approve" (seeding them into search directories and firing confetti) or "Reject".
  • Ecosystem Overview Stats: Real-time platform KPI boxes tracking registered directory vendor counts, active cities, and client inquiry logs.
  • Geographical Database Controller (Cities Hub): Add New Launch City form (Name, State, Tagline, isInternational). Creating a city instantly triggers the generator to seed 159 verified mock vendor profiles across all 53 categories.
  • Support Chat Desk: Panel allowing admins to chat in real-time with clients or vendors.

Part III: Native Mobile App Specifications
PlanMyBaraat

Dual Native Mobile Apps (React Native + Expo)

For the native iOS & Android platforms, we implement two discrete applications sharing a database backend (excluding calculators).

1. Customer Mobile App (expo-customer)

📱 VISUAL MOCKUP: customer_mobile_app_timeline
  • Timeline Script Designer: Interactive chronological vertical timeline. Clients schedule specific steps (e.g. Vintage car entry, pyro fireworks gateway ignition). Clicking checkpoints triggers mock pyrotechnic animations.
  • Safa Turban Selector: A custom picker selecting traditional styles (Rajasthani, Mewari, Punjabi) with quantity counters and sizing configurations (no cost calculations).
  • Song Poll Console: Live voting queue. Guests scan a QR code to vote on DJ truck/band song options and see percentages update instantly.
  • Baraat Route Tracker: Live GPS-style pathway rendering progress, issuing narrow street warning popups, and auto-firing pyrotechnics upon arrival.

2. Vendor Mobile App (expo-vendor)

📱 VISUAL MOCKUP: vendor_mobile_app_inbox
  • Match Leads Queue: Live notifications of incoming client inquiries generated on the Next.js website matching the vendor's operating city and specialty category.
  • Availability Switch: Simple toggler indicating business status (Active, Fully Booked, Inactive) using high-contrast red, yellow, and green status rings.
  • Pricing Manager: Numeric input allowing owners to change starting package prices, updating listing details in the Next.js search directory in real-time.
  • Booking Confirmations: Displays booking details and issues itemized verification receipts.
